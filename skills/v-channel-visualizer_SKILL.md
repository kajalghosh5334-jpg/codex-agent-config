# V-Channel Visualizer

## 定位

由 `output-engine` 在 Q5 五问完成后调用，作为视觉输出的统一决策引擎。
调用方式：`skill: v-channel-visualizer`，传入 Q1 场景和 Q5 输出结构。
不直接面向用户，不输出中间过程，只输出最终视觉渲染指令。

---

## 执行流程总览

```
output-engine Q5 完成
  ↓
V 通道门卫判定（与本 skill 其他模块并行）
  ↓
├─ 深度工程态 → 强路由触发 → 渲染决策 → 输出（含防溢出校验）
├─ 教学/解释类 → 主动触发 → 渲染决策 → 输出
├─ 流程/状态机/对比 → 按复杂度触发 → 渲染决策 → 输出
└─ 轻量交流态 → 禁止触发 → 不渲染，纯文本输出
```

门卫判定与渲染决策均在本 skill 内部完成，不向用户输出中间过程。
触发原因、图类型选择、色系决策全部静默处理。

---

## 门卫判定（触发前必须执行）

### 判定优先级（从高到低）

```
P0 · 教学/解释类                → 无条件触发，跳过所有后续判定
  触发词：讲解XX / 解释XX / 介绍XX / XX是什么 / XX怎么工作
  目的：Visualizer 规范明确要求教学场景必须可视化，优先级最高

P1 · 深度工程态                  → 强触发，不允许跳过
  条件（必须全部满足）：
    A. 内容属于：业务实体 / 数据模型 / 底座设计 / 状态机
    B. 核心链路节点 > 3
  不满足 B 则降级到 P2

P2 · 流程/状态机/对比             → 按复杂度触发
  条件：节点 2-3 或层级 2-3 → 触发
        节点 1 或层级 1 → 不触发（太简单，文字即可）

P3 · 轻量交流态                  → 禁止触发，纯文本输出
  条件：纯概念探讨 / 文案润色 / 轻量 PRD 修改
```

### 门卫输出格式（静默，不向用户展示）

```
V通道判定：[触发 / 禁止]
触发原因：[深度工程态 / 教学场景 / 流程图 / 其他]
不做任何文字说明，直接进入渲染决策或结束。
```

---

## 渲染决策（触发通过后执行）

### 图类型路由表

按 Q1 场景和内容类型选择渲染方式：

```
数据模型 / 数据库设计   → Mermaid erDiagram         禁止手绘 SVG
                        → 节点 > 8 时拆为多张 erDiagram

技术架构 / 系统设计     → Structural SVG             节点 > 8 按模块拆图
                        → 层级 > 4 改用 LR 布局或拆图

教学 / 解释类           → Illustrative SVG           空间隐喻，直觉优先
                        → 可能叠加 HTML stepper（循环/周期类）

流程 / 状态机           → Flowchart SVG              顺序流程用箭头连接
                        → 状态 > 5 改用 HTML stepper

数据对比 / 图表         → Chart.js                   柱状/折线/饼图
                        → ≤ 12 类别禁止自动跳过标签

UI / 界面 mockup        → HTML widget                嵌入 card 容器

地理分布                → D3 choropleth              仅用 CDN 真实拓扑数据
```

### 教学场景特殊路由

```
讲解 LLM / Transformer      → Illustrative SVG（token行 + layer slabs + attention threads）
讲解 TCP / 网络协议          → Flowchart SVG + Illustrative SVG 组合
讲解 事件循环 / 生命周期     → HTML stepper（禁止环形图）
讲解 架构对比（如微服务 vs 单体）→ Structural SVG（并列对比布局）
```

---

## 防溢出规则（不可覆盖，渲染前强制校验）

触发后生成任何图，渲染前必须过以下校验，未通过则停止渲染并拆分/调整：

| 规则 | 限制值 | 违反时处理 | 校验位置 |
|---|---|---|---|
| 单图节点数 | ≤ 8 | 拆为多图，每图标注「图 1/N」 | 渲染前 |
| 纵向层级（TD） | ≤ 4 层 | 改用 LR 布局；仍超出则拆图 | 布局阶段 |
| 单节点文本 | ≤ 30 字 | 截断并在图下方附完整文本 | 文本阶段 |
| Mermaid 色系 | 强制注入 unified head | 无例外，所有 Mermaid 图必须注入 | 生成阶段 |

---

## Mermaid 统一色系 Head

所有 Mermaid 图（erDiagram / flowchart / classDiagram / stateDiagram）必须在渲染指令中包含以下 head，**不可省略，不可修改**：

```
%%{init: {
  'theme': 'base',
  'themeVariables': {
    'background': '#FBF9F6',
    'primaryColor': '#FFFFFF',
    'primaryTextColor': '#191919',
    'primaryBorderColor': '#D1C7BD',
    'lineColor': '#877B6F',
    'secondaryColor': '#F3ECE4',
    'tertiaryColor': '#FBF9F6'
  }
}}%%
```

## SVG / HTML 渲染规范

### 配色体系

9 个颜色梯度，每梯度 7 个层级（50/100/200/400/600/800/900）。层级含义：50=最浅填充，400=主色调，600=强调/描边，800-900=文字。

场景 → 梯度路由：

```
技术架构 / 系统图     → c-blue, c-purple
数据模型 / 状态机     → c-teal, c-green
业务 / 产品           → c-coral, c-pink
风险 / 警告           → c-amber, c-red
中性背景 / 辅助       → c-gray
```

亮色模式（默认）：50 fill + 600 stroke + 800 标题 / 600 副标题

### SVG 布局规则

- viewBox 固定 `0 0 680 H`，H = 最底元素坐标 + 20px，不猜测
- 安全区域：x=40 至 x=640，y=40 至 y=(H-40)
- 背景透明
- 节点间距 ≥ 60px，节点内 padding ≥ 24px
- 同一内容类型节点高度保持一致（单行 44px / 双行 56px）
- 0.5px 描边用于图表边框和连线
- 所有连接线 `<path>` 或 `<polyline>` 必须设置 `fill="none"`

### HTML widget 规则

- 无 `<!DOCTYPE>`、`<html>`、`<head>`、`<body>` 标签——直接输出内容片段
- 无深色/彩色外容器背景（透明，宿主 UI 提供背景）
- 标题 15px/500，副标题 14px/500，正文 13px/400
- 圆角优先 `var(--border-radius-lg)` (12px)
- 图表高度：300px 默认，(bar数量 × 40 + 80)px 水平柱状图

## 工程执行模式附加规则

当 Q1 场景 = 工程执行/代码时，视觉输出额外遵守以下规则（与 output-engine 工程执行10条规则对齐）：

```
修改预算（渲染侧）：
  单次渲染涉及图的数量 ≤ 3
  单张图元素数 ≤ 8（已由防溢出规则覆盖）
  超出时停止，说明原因，等待确认——禁止以「顺便」为由扩展

注意力分配：
  渲染任务作为 Q5 输出的一部分，不干扰主任务目标
  渲染期间禁止处理未授权问题
```

---

## 调用时序

```
output-engine Q5 输出决策阶段：
  1. Q1 场景已确定
  2. Q5 输出结构已确定
  3. 调用 skill: v-channel-visualizer
      ├─ 执行门卫判定
      ├─ 决定是否触发
      ├─ （如触发）选择图类型 + 配色
      ├─ 渲染前过防溢出校验
      └─ 返回最终视觉渲染指令
  4. output-engine 将渲染指令嵌入 Q5 输出
```

v-channel-visualizer 的返回值无需向用户单独展示，直接嵌入 Q5 输出结构中的对应位置。

---

## 禁止行为

```
禁止因「内容不够复杂」跳过教学场景的视觉触发
禁止因「图太大」违反防溢出规则（必须拆分）
禁止用 Mermaid 默认色系替代统一色系 head
禁止在轻量交流态触发任何视觉输出（包括「画一张小图」）
禁止绕过门卫判定直接渲染
禁止将门卫判定过程输出给用户
```
